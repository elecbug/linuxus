# 🔐 Security Policy

## 📌 Reporting a Vulnerability

If you discover a security vulnerability in Linuxus, **do NOT open a public issue**.

Instead, please report it privately via:

- Email: deveb1479@gmail.com

---

## 📋 What to Include

Please include as much detail as possible:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)
- Screenshots or logs (if applicable)

---

## ⏱ Response Policy

- Initial response: within **48 hours**
- Status update: within **3–7 days**

---

## 🔒 Disclosure Policy

- Vulnerabilities will be fixed **before public disclosure**
- After patching, details may be disclosed responsibly

---

## 🚨 Scope

This project includes:

- Web authentication system
- Docker container orchestration
- File system mounts (`homes`, `share`, `readonly`)
- Shell access via web interface

### High-risk areas:

- Authentication bypass
- Unauthorized container access
- File system escape / privilege escalation
- Volume mount misconfiguration

---

## 🙏 Responsible Disclosure

We appreciate responsible disclosure.

Please avoid:

- Publicly disclosing vulnerabilities before a fix
- Exploiting the vulnerability beyond proof-of-concept

---

## 🛡️ Notes

This project is intended for **educational environments**, but security is still taken seriously.

Thank you for helping keep Linuxus secure.
